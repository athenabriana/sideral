## Problem Statement

Personal Fedora atomic desktop built directly on `ghcr.io/ublue-os/silverblue-main:43`. GNOME + tiling-shell session, curated flatpak set auto-installed on first boot, 15-tool mise toolchain (11 language runtimes + 4 CLI tools) declared by default, user dotfiles shipped via `/etc/skel`. Image + dotfiles live in a single monorepo.

> **2026-04-23 status note:** mise toolchain reduced to 12 tools (act, direnv dropped; atuin moved to home-manager). Dotfiles now ship via `/etc/skel/.config/home-manager/home.nix` only (single file). See nix-home spec for the migration.

## Goals

- [ ] Image `ghcr.io/<user>/sideral:latest` built daily by GH Actions (signed with keyless cosign, tagged `:latest`, `:YYYYMMDD`, `:sha-<short>`)
- [ ] Rebase + reboot → GNOME session with 5 extensions active and captured dconf settings applied
- [ ] Custom keybinds active (Ctrl+Alt+T / Super+T → Ptyxis, Ctrl+Shift+Esc → Resources, Ctrl+Alt+Space / Super+. → Smile, Super+Down → minimize)
- [ ] `helium-bin` (from `imput/helium` COPR) installed — browser is in the image, not via flatpak
- [ ] `code` (Microsoft VS Code RPM) installed — editor in image; vscode.repo stays enabled for weekly MS updates
- [ ] First-login user systemd units (`sideral-mise-install.service`, `sideral-vscode-setup.service`) install mise + pre-install `act`/`atuin`/`direnv` + install 3 VS Code extensions (Remote-SSH, Remote-Containers, Azure-Containers)
- [ ] 7 curated flatpaks auto-install on first boot (systemd oneshot, idempotent)
- [ ] `ublue-os-update-services` (inherited from silverblue-main) keeps background auto-update working
- [ ] 15-tool mise toolchain declared in `~/.config/mise/config.toml` via `/etc/skel` (11 language runtimes + 4 CLI tools: act, atuin, direnv, pnpm)
- [ ] `starship` prompt, `atuin` history (Ctrl+R), `direnv` hook activate in every bash shell (host + distrobox) via `/etc/skel/.bashrc`
- [ ] Source Serif 4 + Source Sans 3 installed from Adobe GitHub at image-build time
- [ ] `/etc/os-release` rewritten: `ID=sideral`, `NAME="Sideral OS"`
- [ ] Zero reference to `hyprland`, `ags`, `astal`, `waybar`, `rofi`, `wlogout`, `swaync` in the repo tree (these were artifacts of the earlier Hyprland direction)
- [ ] Zero reference to `bluefin`, `bluefin-dx`, `projectbluefin` in **runtime artifacts** (build-time comments like "borrowed pattern from bluefin" are acceptable)

## User Stories

### P1: Rebase-and-go GNOME ⭐ MVP

**Story**: Rebase any Fedora-atomic host to `sideral:latest`, reboot, log in — land in a fully configured GNOME + tiling-shell session with no manual setup.

**Acceptance**:

1. **ATH-01** — A push to `main` triggers CI that builds, signs, and pushes `:latest`, `:YYYYMMDD`, and `:sha-<short>` tags.
2. **ATH-02** — `rpm-ostree rebase ostree-unverified-registry:ghcr.io/<user>/sideral:latest` succeeds and stages the new deployment.
3. **ATH-03** — After reboot, GDM presents a GNOME session (Wayland).
4. **ATH-04** — At first login, exactly these 4 extensions are enabled without manual action:
   `appindicatorsupport@rgcjonas.gmail.com`, `dash-to-panel@jderose9.github.com`, `tilingshell@ferrarodomenico.com`, `rounded-window-corners@fxgn`.
   *(Updated 2026-05-01: bazaar-integration removed alongside the Bazaar→GNOME-Software swap.)*
5. **ATH-05** — Captured dconf defaults apply: dash-to-panel icons/position, tiling-shell layouts+autotiling+snap-assist, rounded-corners radius 12px+skip-libadwaita.
6. **ATH-06** — Window focus follows mouse without click (sloppy focus, `auto-raise=false`).
7. **ATH-07** — Custom keybinds work: `Ctrl+Alt+T` / `Super+T` → Ptyxis; `Ctrl+Shift+Esc` → Resources; `Ctrl+Alt+Space` / `Super+.` → Smile; `Super+Down` → minimize.
8. **ATH-08** — `rpm-ostree status` shows sideral as current with the previous deployment preserved as rollback target.

**Test**: Fresh VM, rebase, reboot, log in. `gnome-extensions list --enabled` outputs the 4 UUIDs.

---

### P2: Flatpak auto-install on first boot

**Story**: 7 curated flatpaks install themselves on first boot so the desktop is usable without manual setup.

**Acceptance**:

1. **ATH-09** — A oneshot systemd unit installs every ref in `/etc/flatpak-manifest` on first boot.
2. **ATH-10** — Already-installed refs are skipped (idempotent).
3. **ATH-11** — After success, the service writes a sentinel file and does not re-run on subsequent boots.
4. **ATH-12** — If the network is unavailable, the service logs the failure and exits non-blocking.
5. **ATH-13** — `flatpak list --app` shows all 7 refs after first-boot completion.

**Test**: Fresh rebase with network, open the Activities overview — Flatseal, Warehouse, Extension Manager, Podman Desktop, DistroShelf, Resources, and Smile are all launchable.

---

### P2: VS Code ready-to-code

**Story**: Launching Code from the app grid after first login gives a working editor with Remote-SSH, Remote-Containers, and Azure-Containers already installed.

**Acceptance**:

1. **ATH-14** — `/etc/yum.repos.d/vscode.repo` is shipped enabled; `code` RPM layered from Microsoft's repo.
2. **ATH-15** — `sideral-vscode-setup.service` runs on first login, installs three extensions, writes `~/.cache/sideral/vscode-setup-done` marker.
3. **ATH-16** — On subsequent logins, the service skips (condition fails).
4. **ATH-17** — `code --list-extensions` shows all three bundled UUIDs after first login completes.
5. **ATH-18** — `rpm-ostree upgrade` pulls new VS Code versions directly from Microsoft's repo (repo stays enabled).

---

### P2: Dotfile iteration

**Story**: Edit configs live, round-trip through the repo, and have the next image ship updated defaults.

**Acceptance**:

1. **ATH-19** — `just capture-home` rsyncs `~/.config/mise/` (and any future tracked dirs) into `home/.config/`.
2. **ATH-20** — `just apply-home` rsyncs `home/.config/` onto `~/.config/` (overwrites tracked files only, leaves untracked alone).
3. **ATH-21** — `just diff-home` prints a diff between repo and live `~/.config`.
4. **ATH-22** — On a new image release, updated `/etc/skel/.config/*` defaults are available to new users automatically and to the existing user via `just apply-home`.

**Test**: Edit `~/.config/mise/config.toml`, run `just capture-home`, `git diff` shows the change.

---

### P3: mise toolchain preloaded

**Acceptance**:

1. **ATH-23** — `/etc/skel/.config/mise/config.toml` declares 15 tools grouped by context (JS/TS, Python, JVM, Systems, Mobile, Shell UX, CI) with symbolic versions where possible.
2. **ATH-24** — `/etc/skel/.bashrc` auto-activates starship, mise, atuin, and direnv in every interactive bash shell (host + distrobox).
3. **ATH-25** — `trusted_config_paths = ["/"]`, `not_found_auto_install = true`, `idiomatic_version_file_enable_tools` cover `.nvmrc` / `.python-version` / etc. in cloned repos.
4. **ATH-26** — `sideral-mise-install.service` eagerly installs `act`, `atuin`, `direnv` on first login (because their shell hooks fire before lazy install could).
5. **ATH-27** — First use of any other declared tool (e.g. `node -v`) triggers lazy install of just that tool; untouched tools never download.

## Edge Cases

- If an RPM package rename breaks `dnf install`, CI fails loudly (don't ship broken image).
- If extension build-time download from extensions.gnome.org fails, CI fails.
- If `bootc container lint` fails at end of build, CI does not push the image.
- Existing user rebasing to sideral keeps the previous deployment as rollback target.

## Success Criteria

- [ ] Fresh VM rebase → GNOME + tiling-shell session within ~30 seconds of login, 5 extensions active
- [ ] First-boot flatpak service completes within 5 minutes on a reasonable connection; 7 apps installed
- [ ] CI build under 15 min (RPM installs + 2 extension zips + 2 font zips from GitHub)
- [ ] Image size within ~1 GB of `silverblue-main:43` (Helium ~350 MB + VS Code ~300 MB + docker-ce stack ~200 MB + ~15 small RPMs + 2 extension zips + 2 font packs)
- [ ] Previous deployment preserved as rollback on first rebase
