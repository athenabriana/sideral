# Kubernetes-from-Fedora-main layer.
#
# kubectl is NOT in Fedora 43 main (no `kubernetes-client` package);
# it's installed by kubectl-install.sh in this same module dir from
# the pkgs.k8s.io persistent repo (sideral-kubernetes ships the .repo
# file). Both kind and helm ARE in Fedora main as of F43, so they go
# through the orchestrator's per-module install loop.
#
#   kind  — Kubernetes-IN-Docker. Spins up real K8s clusters as
#           podman containers (no VM). Default backend for Podman
#           Desktop's "Create new cluster". Pair with the
#           KIND_EXPERIMENTAL_PROVIDER=podman env var that
#           sideral-shell-ux now ships in /etc/profile.d/.
#   helm  — Kubernetes package manager. Templates YAML manifests
#           into "charts" (`helm install foo bitnami/postgresql`
#           replaces ~15 `kubectl apply`s). Podman Desktop's Helm
#           panel shells out to this binary.
